170    location: uri that identifies from where to read data in the specified format.
171    enforce_schema_from_table: if we want to enforce the table schema or not, by

174        read from a file system nor from a table, but rather from a sql query instead.

176        schema).
177    schema_path: path to a file with a representation of a schema of the input (e.g.,
178        Spark struct type schema).
179    with_filepath: if we want to include the path of the file that is being read. Only
180        works with the file reader (batch and streaming modes are supported).
181    options: dict with other relevant options according to the execution
182        environment (e.g., spark) possible sources.
183    calculate_upper_bound: when to calculate upper bound to extract from SAP BW or not.
184    calc_upper_bound_schema: specific schema for the calculated upper_bound.
185    generate_predicates: when to generate predicates to extract from SAP BW or not.
186    predicates_add_null: if we want to include is null on partition by predicates.

194    db_table: Optional[str] = None
195    location: Optional[str] = None

273            source data to identify the rows failing the DQ validations. Note: only one
274            of tbl_to_derive_pk or unexpected_rows_pk arguments need to be provided. It

282            mandatory, but still recommended.
283        gx_result_format - great expectations result format. Default: "COMPLETE".
284    ´   tag_source_data - when set to true, this will ensure that the DQ process ends by
285            tagging the source data with an additional column with information about the
286            DQ results. This column makes it possible to identify if the DQ run was
287            succeeded in general and, if not, it unlocks the insights to know what
288            specific rows have made the DQ validations fail and why. Default: False.
289            Note: it only works if result_sink_explode is True, gx_result_format is

291            you specify tag_source_data as True) and tbl_to_derive_pk or

293        store_backend - which store_backend to use (e.g. s3 or file_system).
294        local_fs_root_dir - path of the root directory. Note: only applicable for

482@dataclass
483class ReconciliatorSpec(object):
484    """Reconciliator Specification.
485
486    metrics: list of metrics in the form of:
487        [{
488            metric: name of the column present in both truth and current datasets,
489            aggregation: sum, avg, max, min, ...,
490            type: percentage or absolute,
491            yellow: value,
492            red: value
493        }].
494    recon_type: reconciliation type (percentage or absolute). Percentage calculates
495        the difference between truth and current results as a percentage (x-y/x), and
496        absolute calculates the raw difference (x - y).
497    truth_input_spec: input specification of the truth data.
498    current_input_spec: input specification of the current results data
499    truth_preprocess_query: additional query on top of the truth input data to
500        preprocess the truth data before it gets fueled into the reconciliation process.
501        Important note: you need to assume that the data out of
502        the truth_input_spec is referencable by a table called 'truth'.
503    truth_preprocess_query_args: optional dict having the functions/transformations to
