238    """Specification of an algorithm input.
239
240    This is very aligned with the way the execution environment connects to the sources
241    (e.g., spark sources).
242
243    spec_id: spec_id of the input specification read_type: ReadType type of read

246    sftp_files_format: format of the files (csv, fwf, json, xml...) in a sftp
247        directory.
248    df_name: dataframe name.
249    db_table: table name in the form of <db>.<table>.
250    location: uri that identifies from where to read data in the specified format.
251    enforce_schema_from_table: if we want to enforce the table schema or not, by
252        providing a table name in the form of <db>.<table>.
253    query: sql query to execute and return the dataframe. Use it if you do not want to
254        read from a file system nor from a table, but rather from a sql query instead.
255    schema: dict representation of a schema of the input (e.g., Spark struct type
256        schema).
257    schema_path: path to a file with a representation of a schema of the input (e.g.,
258        Spark struct type schema).
259    with_filepath: if we want to include the path of the file that is being read. Only
260        works with the file reader (batch and streaming modes are supported).
261    options: dict with other relevant options according to the execution
262        environment (e.g., spark) possible sources.
263    calculate_upper_bound: when to calculate upper bound to extract from SAP BW or not.
264    calc_upper_bound_schema: specific schema for the calculated upper_bound.
265    generate_predicates: when to generate predicates to extract from SAP BW or not.
266    predicates_add_null: if we want to include is null on partition by predicates.

365            tagging the source data with an additional column with information about the
366            DQ results. This column makes it possible to identify if the DQ run was
367            succeeded in general and, if not, it unlocks the insights to know what
368            specific rows have made the DQ validations fail and why. Default: False.
